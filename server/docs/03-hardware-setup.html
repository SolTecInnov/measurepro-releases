Hardware Setup Guide

Zero ambiguity. Follow this sequence every time.

CRITICAL: Startup order matters. If sensors are powered before the app is open and connected, you will get silent failures. Follow the sequence below exactly.

1. Mounting Positions

Vertical Clearance Laser (RSA / LDM71)

  • Position: Vehicle rooftop, centred on the longitudinal axis
  • Orientation: Beam pointing vertically upward (within ±2°)
  • Height above ground: Measure and record — this becomes your ground reference value in Settings
  • Mount type: Rigid, vibration-isolated bracket — no flex or rattle
📷 Photo placeholder: Correct laser roof mount — centred, upright, cables secured
✗ WRONG
Laser tilted more than 2° from vertical. Strong side-lobe returns will produce false readings 0.3–0.8 m lower than actual clearance.
✓ RIGHT
Laser perfectly vertical, confirmed with a spirit level. Ground reference calibrated to actual rooftop height.

GNSS Antenna (Duro)

  • Position: Rooftop, clear 360° sky view — no vehicle body within 15° elevation
  • Separation from laser: Minimum 30 cm to avoid multipath interference
  • Cable: Keep RF cable away from inverters and power wiring
📷 Photo placeholder: Duro antenna placement on vehicle roof with correct separation from laser

Lateral Lasers (Left / Right)

  • Position: Bumper or side-rail mount, beam pointing horizontally outward (perpendicular to travel direction)
  • Height above ground: [FILL IN — match to obstacle type being measured]
  • Offset from vehicle edge: Measure and enter in Settings → Lateral Laser Settings

Rear Overhang Laser

  • Position: Rear of vehicle/trailer, beam pointing straight rearward along centreline
  • Use case: Wind blade tip monitoring during dry runs
  • Max range configured: Up to 80 m

4. Startup Sequence

  1. 1
    Power on Duro GNSS receiver first — allow 60–90 seconds to acquire satellites and establish RTK fix before launching the app.
  2. 2
    Power on all lasers — vertical, lateral, and rear if fitted. Confirm power indicator LEDs are active.
  3. 3
    Connect all USB serial cables to the controller device. Check Device Manager (Windows) or ls /dev/tty* (Linux/macOS) to confirm ports are enumerated.
  4. 4
    Launch MeasurePRO in Chrome or Edge (version 89+). Do not use Safari or Firefox — Web Serial API is not supported.
  5. 5
    Connect vertical laser — Settings → Laser → Select Port → Connect. Confirm live readings appear in the Measurement card.
  6. 6
    Connect GPS source — Settings → GPS. Connect Duro (TCP), USB GPS, or Bluetooth GPS. Confirm the GPS indicator turns green.
  7. 7
    Connect lateral / rear lasers if required — Settings → Multi-Laser. Assign each port (Left / Right / Rear).
  8. 8
    Run ground reference calibration — aim laser at ground, capture 3 readings, confirm ground reference value in Settings. (See Doc 05 — Field Operation SOP.)
  9. 9
    Start LiDAR Windows service (if fitted) — run MeasurePROLidarService.exe, confirm connection in app's LiDAR panel.
  10. 10
    Final sensor validation — walk a known-clearance structure and confirm readings are within expected range. Do not begin survey until validated.
✗ WRONG
Opening the app and trying to connect the laser before Duro has acquired a GPS fix. GPS and laser state will be mismatched and the first POIs will have no coordinates.
✓ RIGHT
Duro fix confirmed (green RTK indicator) before app is launched. All sensors connected before leaving the yard.

5. On-Site Configuration Checks

  • Verify ground reference value has not changed from yard calibration (laser height above ground must not have shifted during transit)
  • Check ignoreAbove / ignoreBelow thresholds are set correctly for the route profile (Settings → Laser Thresholds)
  • Confirm amplitude filter levels are appropriate for conditions (weak signal in bright sunlight may need lower threshold; rain may need higher threshold to reject false returns)
  • Verify rear overhang threshold is set for the load being transported (default max 80 m)
  • Confirm minimum curve radius setting matches your vehicle specification (default 15 m)

Common mistake: Selecting RSA protocol for an LDM71 device (or vice versa). RSA uses 3-byte binary frames; LDM71 outputs ASCII decimal. The symptom is exactly 0.000 m on every reading. Fix: disconnect laser → change protocol → reconnect.

7. Amplitude Filter

The amplitude filter rejects laser returns whose reflected signal strength (dBm) falls below a configurable minimum. This prevents weak/noisy returns from polluting survey data.

ConditionActionWhy
Rain / DustIncrease minimum amplitudeReject false rain-drop and dust-particle returns at short range
Dark asphalt / max range targetsDecrease minimum amplitudeAccept weaker but valid returns from low-reflectance surfaces
Bright direct sunlight (LDM71)Decrease minimum amplitudeSolar saturation lowers return dBm; threshold must accommodate this
Normal conditionsDefault (factory setting)Balanced rejection / acceptance

Configure: Settings → Laser → Amplitude Filter
