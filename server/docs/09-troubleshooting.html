Troubleshooting Matrix

Problem → Cause → Action. Find your issue, follow the action. No guessing.

Problem Likely Cause Action
Vertical Clearance Laser
No laser readings in app Serial port not connected or wrong port selected Settings → Laser → disconnect, select correct COM port, reconnect. Confirm USB cable is plugged in directly (not via hub).
Laser shows -- constantly Beam hitting sky / no return; ignoreAbove too low; amplitude filter too strict; DE02 error 1. Aim at a solid surface within 15 m. 2. Check ignoreAbove setting. 3. Lower amplitude filter threshold. 4. If outside, ensure laser faces upward with clear path to overhead objects.
Readings jump ±0.5 m randomly Laser mount vibrating; cable connector loose; weak return signal Tighten mount bolts. Check USB connector is firmly seated. Increase amplitude filter minimum to reject weak returns. Reduce vehicle speed over rough road.
All readings exactly 0.000 m Wrong laser protocol selected Settings → Laser Protocol — confirm RSA vs LDM71 matches hardware. RSA = 3-byte binary; LDM71 = ASCII. Reconnect after changing.
Readings consistently 0.3–0.8 m too low Ground reference not set; laser tilted >2° Re-run ground reference calibration. Check laser vertical alignment with spirit level.
Laser connects then immediately disconnects Serial port taken by another process; USB hub power issue Close any other serial terminal software. Replug to direct USB port. Windows: check Device Manager for port conflicts.
GPS / GNSS
GPS indicator stays red / no fix Duro not powered; wrong IP/port; TCP bridge not running Confirm Duro LED is active. Check TCP bridge server is running on the controller laptop. Verify IP and port in Settings → GNSS → Duro Config.
GPS position shows correct road but shifts 20–30 m RTK fix lost — degraded to SBAS or autonomous mode Check RTK indicator in GNSS panel. If degraded, wait for Duro to re-acquire RTK baseline. Note the section in survey comments. Do not use for precision engineering until RTK restored.
Duro connected but USB GPS also showing Both sources active; USB GPS should be silenced by priority logic Duro takes priority automatically when it has sent data within 5 seconds. If both are active simultaneously, disconnect USB GPS in Settings → GPS Sources.
GPS track shows vehicle teleporting GNSS multipath or interference near large metal structures Expected near bridges and under power pylons. The app will record the jump but filter outliers during export processing. Note the location.
No GPS in tunnel No satellite signal underground Expected. Record entry and exit GPS coordinates before/after the tunnel. The export will show a straight-line interpolation through the tunnel. Note in deliverable.
Hesai LiDAR (Companion Service)
LiDAR service not connecting in app Service not running; wrong port; Windows firewall 1. Confirm MeasurePROLidarService.exe is running. 2. Default port 17777 — check app LiDAR settings match. 3. Add firewall exception for ports 17777–17787. 4. Check service logs in C:\MeasurePRO\Logs\.
LiDAR data streaming but no point cloud Capture not started; disk full Press "Start Static Capture" or "Start Segment Capture" in app. Confirm C:\MeasurePRO\Captures\ has free space (>10 GB recommended per hour).
Service crashes mid-capture Memory overload; UDP packet loss Restart service. Capture to this point is saved. Restart the session as a new capture segment and note the gap. Ensure no other UDP-heavy applications running on the LiDAR network interface.
App / PWA
App not loading (blank screen) Service worker cache stale; browser not supported Hard refresh: Ctrl+Shift+R (Windows) / Cmd+Shift+R (Mac). If still blank: Settings → Clear site data in Chrome. Use Chrome or Edge only — Safari/Firefox do not support Web Serial API.
"Cannot connect to serial port" error Browser does not have permission; wrong browser Must use Chrome or Edge version 89+. When connecting for first time, browser shows a port selection dialog — user must manually grant access. If permission was previously denied, reset in Chrome → Settings → Privacy → Site Settings → Serial.
Survey data missing after app crash IndexedDB write behind on flush App flushes to IndexedDB every 30 seconds. Reopen app — survey should restore from last flush. If data lost: last resort is to re-survey the section from the last confirmed POI.
Cloud sync not completing (spinner stays) No internet connection; Firebase quota Data is safe in IndexedDB locally. Connect to internet and sync will resume automatically. Check Firebase console for quota warnings if persistent.
POIs appear in wrong location on map GPS was degraded at time of capture; CRS mismatch in export Check GPS accuracy indicator at time of capture (visible in survey log). For export CRS issues: re-export with the correct CRS for the client's GIS system.
Road Profiling & GNSS
Road Profile panel not appearing Module not enabled; wrong GPS source Settings → Road Profile → Enable Road Profile Recording. Requires Duro with RTK Fixed fix — USB GPS, Bluetooth, and browser geolocation do not support Road Profiling.
Grade values look wrong or jump RTK fix not achieved; fast speed over short segment Confirm RTK Fixed (green indicator, not Float). Grade accuracy improves at constant speed. K-factor spikes on short crest/sag curves are normal.
Banking / cross-slope reads 0° constantly IMU not available; USB or BT GPS in use Cross-slope requires the Duro's IMU output. Confirm the Duro is connected and providing roll data — check the raw GNSS panel for IMU roll field.
Voice Commands
Microphone icon missing / voice control not available Settings not enabled; browser permission denied Settings → Voice Commands → Enable Voice Assistant. Grant microphone permission in browser (click lock icon in address bar → Microphone → Allow). Reload page.
Commands not recognized No internet; wrong language; background noise Voice recognition requires internet. Confirm Settings → Voice Commands → Language matches your speech. Reduce background noise. Use Chrome or Edge — Firefox has limited support; Safari is not supported.
Voice notes won't save Microphone permission denied; storage full Voice notes use a separate local recording — check microphone permission. Check available device storage: Survey Manager → Settings → Storage. Voice notes are stored in IndexedDB, not cloud.
Multi-Laser (Lateral / Rear)
Left or right lateral laser not reading Port not assigned; protocol mismatch Settings → Multi-Laser → assign each laser to its USB port. Protocol must be soltec-old (19200 baud, 7E1). USB adapters must be individually labelled — label confusion is the most common cause.
Lateral width reading is implausible (too wide/narrow) Vehicle offset not configured; single vs dual mode mismatch Settings → Lateral Laser Settings → confirm vehicle half-width offsets and single/dual laser mode are correct.
Rear overhang alarm triggering at rest Threshold set too low; blade measurement not started yet Check rear overhang threshold in Settings — default 80 m. If alarm fires before blade is loaded, confirm the laser is reading the actual blade end, not a nearby wall.
