Automatic vs Manual Steps

Fully Automatic (no action needed)

  • GPS coordinate attachment to every POI
  • Photo capture and POI linking
  • Chainage calculation along the route
  • Grade (%) and K-factor calculation
  • Banking / cross-slope recording from Duro IMU
  • Curve radius calculation from GPS trajectory
  • Amplitude filtering (weak returns rejected)
  • Ground reference subtraction from raw measurements
  • IndexedDB flush every 30 seconds (crash recovery)
  • Cloud sync on survey close

Manual / Operator Decision

  • POI type selection (bridge vs overpass vs wire etc.)
  • Adding text notes to POIs
  • Flagging anomalies (re-check required, access issue)
  • CRS selection for engineering exports (WGS84, MGA zone, etc.)
  • Resampling interval (full density vs fixed interval)
  • Altitude mode (raw GPS / corrected / all)
  • LiDAR capture start / stop
  • Review of clearance anomaly flags before delivery

Export Process — Step by Step

  1. Open the Survey Manager (survey list tab in the app).
  2. Select the completed survey and tap "Export".
  3. The Survey Export Dialog opens. Configure:
    • Format — select one or multiple: CSV, GeoJSON, Shapefile, DXF, LandXML
    • CRS — WGS84, Web Mercator, or a specific MGA zone (or custom EPSG code)
    • Altitude mode — raw GPS altitude, corrected, or export all variants
    • Resampling — full density (every recorded sample) or fixed interval (e.g. every 5 m)
  4. Tap Export. The browser downloads a ZIP file immediately.
  5. The ZIP contains: all selected format files + metadata.json (calibration provenance, operator, date, app version).
  6. The ZIP is re-importable — drag it back into MeasurePRO to restore the survey for review or re-export.

Expected processing time: <5 seconds for surveys up to 10,000 POIs / 50,000 GPS samples. Very large surveys (>100,000 records) may take 15–30 seconds.

If You See This → Do This

Export ZIP is empty / 0 bytes
Survey was not closed properly before export. Reopen the survey, tap Close Survey, wait for sync confirmation, then re-export.
GeoJSON has all POIs at coordinates 0,0
GPS was not connected during capture. Check that the GPS indicator was green during the survey. POIs captured without GPS cannot be geo-referenced retroactively.
CSV shows height values much higher than expected
Ground reference was not set or was set to zero. The raw laser reading includes the vehicle roof height. Recalibrate ground reference in Settings and re-export — the export uses stored calibration values if the survey was recalibrated post-capture.
Shapefile import fails in ArcGIS ("invalid geometry")
Confirm the CRS selected during export matches the target project CRS. Shapefile projection is embedded in the .prj file. Re-export with correct MGA zone or WGS84 as required.
LandXML import fails in Civil 3D ("unrecognised format")
Civil 3D requires LandXML 1.2 format. MeasurePRO exports LandXML 1.2 by default. If the issue persists, check the Civil 3D version — older versions may not support all alignment attributes. Try DXF as an alternative.
Road profile export has large gaps in chainage
GNSS signal was lost during that segment. The buffer continued recording but without coordinates. The gap will appear in the GeoJSON trace. Note it in the deliverable and consider a re-survey of the affected section.
Re-import of ZIP fails ("incompatible format")
Older surveys (pre-v2.0 file format) may not re-import cleanly. These must be exported from the original survey record — they cannot be reconstructed from the ZIP alone. The app will display the format version in the import dialog.
