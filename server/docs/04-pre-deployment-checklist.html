Pre-Deployment Checklist

Print this. Laminate it. Complete it every time without shortcuts.

🖨️ This page is print-optimised. Use Ctrl+P / Cmd+P → Print. Checkboxes will appear correctly on paper.
Survey ID: _________________________    Date: _________________________    Operator: _________________________
Section A — Before Leaving the Yard
  • Controller device battery / power Laptop/tablet charged >80% or connected to vehicle power. Sleep/hibernate disabled.
  • Duro GNSS powered and RTK fix acquired Green RTK indicator in app. Allow 60–90 seconds after power-on. Do not leave without RTK.
  • All laser(s) powered and connected Vertical + lateral + rear (if fitted). Power LEDs active. Serial ports enumerated.
  • Ground reference calibrated Aim laser at ground, capture 3 readings, confirm value in Settings → Laser Thresholds. Record value: _______ m
  • ignoreAbove / ignoreBelow thresholds set Configured for today's route clearance range. Typical: ignoreBelow = 0.5 m, ignoreAbove = 12 m.
  • Amplitude filter level confirmed Check signal strength on a known target at typical survey range. Adjust filter if needed.
  • Camera(s) tested Front, left, right, and rear positions assigned in Settings → Multi-Camera. Test capture for each active position.
  • Storage capacity IndexedDB storage <70% full. If low, export and clear completed surveys before departing.
  • App and firmware versions confirmed MeasurePRO version: _______. Duro firmware: _______. LiDAR service version (if fitted): _______.
  • LiDAR Windows service running (if fitted) MeasurePROLidarService.exe running. WebSocket connected (port 17777). Mock mode: OFF.
  • Vehicle dimensions entered Width, height, wheelbase, and overhang values confirmed in vehicle profile. Curve radius threshold set.
  • Rear overhang threshold set (if wind blade run) Max threshold configured for blade length. Alert levels: warning and critical confirmed. Value: _______ m
Section B — On-Site, Before First Pass
  • Laser mount alignment checked Verify laser has not shifted during transit. Use spirit level. Re-calibrate ground reference if shifted.
  • Calibration validation pass Drive under a known-clearance structure (bridge, gantry). Confirm reading matches reference measurement within ±50 mm.
  • GPS time sync Device clock synced to GPS time. Verify in GNSS status panel — time offset <1 second.
  • GNSS accuracy indicator RTK fix confirmed (green). HDOP <2.0. If only SBAS available, note in survey comments.
  • New survey created in app Survey name, road name, and operator recorded. Survey ID: _________________________
  • Road profile recording mode selected GNSS profile recording enabled if engineering grade output is required.
  • Convoy Guardian / Permitted Route loaded (if applicable) Route file imported. All convoy vehicles confirmed connected.
Section C — Post-Survey (Before Leaving Site)
  • Survey closed and synced Tap "Close Survey" in app. Wait for sync confirmation (Firebase cloud icon turns green). Do not power off until synced.
  • Coverage review Check route trace on map — no unexplained gaps or missing segments.
  • POI count reasonable Review POI list — no obvious misses or duplicates from calibration passes.
  • LiDAR capture stopped cleanly (if fitted) Confirm capture ended via HTTP API. LAZ file saved to C:\MeasurePRO\Captures\.
