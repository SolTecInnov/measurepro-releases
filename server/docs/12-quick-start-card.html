Print, laminate, put it in the kit case. If this isn't perfect, the whole system feels clunky.

🖨️ Print this page. Landscape orientation → Scale to fit. Laminate. Punch a hole in the corner and attach to the equipment case with a cable tie.

Laminated Card — Both Sides

▶ FRONT — Setup (do this every time)
1
Power on Duro GNSS Wait 60–90 sec for RTK fix (green LED)
2
Power on all lasers Vertical + lateral + rear. Confirm power LEDs.
3
Open MeasurePRO in Chrome/Edge NOT Safari. NOT Firefox. Chrome or Edge only.
4
Connect laser → confirm readings Settings → Laser → Select Port → Connect. Look for stable numbers in Measurement card.
5
Connect GPS → confirm green indicator Settings → GPS. Duro TCP first; USB GPS fallback.
6
Calibrate ground reference Aim laser at ground → 3 readings → confirm value in Settings
7
Create new survey → start capture New Survey → name → tap Start. Drive validation pass first.
Keyboard shortcuts: Space = capture POI  |  [ = left  |  ] = right  |  ' = rear overhang  |  Alt+3 = Road Profile logging
Road Profile (Duro only) — RTK Fixed → Settings → Road Profile → Enable → Alt+3
Voice Commands — "log measurement" | "start logging" | "what is my height" | "take photo"
Laser protocols — SolTec/RSA/LDM71 = 115,200 8N1  |  Lateral/Rear = 19,200 7E1
▶ BACK — Common Errors + Fixes
Laser shows -- constantly
→ Aim at solid surface. Check ignoreAbove setting. Lower amplitude threshold.
Cannot connect to serial port
→ Use Chrome/Edge. Grant permission when browser asks. Try a different USB port directly (not hub).
GPS indicator stays red
→ Check Duro is powered and TCP bridge is running. Wait 90 sec after power-on.
Readings 0.3–0.8 m too low
→ Re-run ground reference calibration. Check laser is vertical (spirit level).
App blank screen
→ Hard refresh: Ctrl+Shift+R. If still blank, clear site data in Chrome settings.
LiDAR not connecting
→ Start MeasurePROLidarService.exe. Add firewall exception for port 17777.
Survey data gone after crash
→ Reopen app — data restores from last 30-sec flush. Check Survey Manager for partial survey.
POIs at wrong location on map
→ GPS was degraded at capture time. Note in deliverable. Re-survey if precision required.

Emergency Recovery (If Everything Fails)

  1. Do NOT power off — data is still in memory
  2. Open a new browser tab — load the app fresh (data in IndexedDB will persist)
  3. If app won't load at all: note the GPS coordinates manually (phone maps)
  4. Document the time of failure — this determines what data was captured vs lost
  5. Restart the session as a new survey segment from the last confirmed POI
  6. Report the incident with timestamp to your Level 2/3 operator before delivering the survey
