🧩 1. Multi-Worker Architecture (Mandatory)

Important:

💾 2. IndexedDB: Zero-Lag Writing Strategy

Implement:

A. Batching system
	•	Maintain an in-memory queue (RAM)
	•	Batch size target: 20–50 items
	•	Writes occur:
	•	every X seconds (ex: 500–800ms) OR
	•	when queue hits batch size
	•	Use readwrite transactions ONLY in orchestrator
	•	NEVER open more than one transaction at a time

B. Memory Guard
	•	Max RAM queue: 100–150 POI events
	•	If queue exceeds limit:
	•	temporarily throttle reading workers
	•	ensure REAL-TIME always wins
	•	flush aggressively in larger batches

C. Crash-Safe Writes
	•	Every batch must include:
	•	POI type
	•	GPS snapshot
	•	laser measurement
	•	compressed image
	•	timestamp links
	•	timelapse/video pointer (frame # or time)

All grouped under a single indexed object, never separated.
