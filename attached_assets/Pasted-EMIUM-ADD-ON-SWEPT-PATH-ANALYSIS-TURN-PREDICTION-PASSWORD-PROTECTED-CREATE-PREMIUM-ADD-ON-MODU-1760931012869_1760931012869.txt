PART 2: TURN DETECTION & ROAD GEOMETRY ANALYSIS
STEP 4 - IMPLEMENT ROAD SEGMENTATION: Create async function segmentRoadSurface with parameter videoFrame. Load DeepLab model if not already loaded using loadSegmentationModel. Run model prediction: const segmentation = await model.segment(videoFrame). Extract segmentation mask with classes: road surface, road edge, lane marking, obstacle, background. Convert mask to binary road/non-road using threshold. Apply morphological operations using OpenCV to clean mask: erosion then dilation. Extract contours of road area using cv.findContours. Return roadMask and roadContours for turn detection. This runs in parallel with existing COCO-SSD detection without interference.
STEP 5 - CREATE TURN DETECTOR CLASS: Create class TurnDetector with constructor taking calibrationData parameter. Implement method detectTurn with parameter videoFrame. Call segmentRoadSurface to get road mask. Extract left boundary by finding leftmost road pixels at each y-coordinate. Extract right boundary by finding rightmost road pixels at each y-coordinate. Fit polynomial curve to boundary points using Math.js polynomial regression of degree 2. Calculate curvature at multiple points using formula: curvature = abs(second derivative) divided by (1 + first derivative squared) to power 1.5. Average curvatures to get mean curvature. Calculate radius as 1 divided by curvature. Classify as turn if radius less than 100 meters and curvature greater than 0.01. Calculate turn angle by measuring change in tangent direction between entry and exit. Determine turn type by checking if curvature is positive (left turn) or negative (right turn). Return turnData object with detected boolean, radius number, angle number in degrees, turnType string, leftBoundary array, rightBoundary array, entryPoint object, exitPoint object.
STEP 6 - IMPLEMENT VANISHING POINT DETECTION: Create function detectVanishingPoint with parameter roadMask. Apply Canny edge detection: cv.Canny(roadMask, edges, 50, 150). Detect lines using Hough Line Transform: cv.HoughLinesP(edges, lines, 1, Math.PI/180, 50, 50, 10). Filter lines to keep only those with angles between 60 and 120 degrees (relatively vertical). Extend all lines to find intersection points. Cluster intersection points to find most common intersection area. Calculate vanishing point as centroid of largest cluster. Use vanishing point y-coordinate to calculate camera pitch angle. Store vanishing point for perspective correction in distance calculations. Return vanishingPoint object with x y coordinates and pitchAngle in degrees. This complements existing camera calibration.
STEP 7 - CREATE ROAD WIDTH MEASUREMENT: Create function measureRoadWidthAlongCurve with parameters turnData and calibrationData. Sample turn curve at regular intervals of 5 meters using polynomial curve from turnData. At each sample point find left boundary pixel and right boundary pixel from turnData boundaries. Calculate pixel distance between boundaries. Convert pixel distance to real-world meters using camera calibration focal length and vanishing point for perspective correction. Formula: realDistance = pixelDistance times distanceToGround divided by focalLength times cosine of viewAngle. Detect obstacles from road mask that encroach on width. Calculate effective width as measured width minus obstacle intrusions. Store all width measurements with their positions along curve. Find minimum width location. Return widthProfile array with objects containing position, width, effectiveWidth, hasObstacles, and minimumWidth object with value and location.

STEP 9 - IMPLEMENT ARTICULATION JOINTS: Create class ArticulationJoint with constructor taking parameters upstreamSegment and downstreamSegment. Add properties: flexAngle number representing maximum articulation in degrees defaulting to 45, currentAngle number as 0, jointType string as ‘fifth_wheel’ or ‘king_pin’ or ‘pivot’. Implement method calculateArticulationAngle that computes angle between connected segments: get upstream rear position and angle, get downstream front position, calculate vector from upstream rear to downstream front, calculate angle of this vector, compute difference between upstream angle and vector angle, constrain result to be between negative flexAngle and positive flexAngle, return constrained angle. Implement method applyFlexLimit with parameter desiredAngle that clamps angle to allowable range based on jointType: fifth wheel 45 degrees, king pin 60 degrees, pivot 45 degrees.
STEP 10 - CREATE STEERABLE AXLE COMPENSATION: Create class SteerableAxleSystem with constructor taking parameters axleCount and segmentLength. Add properties: maxSteeringAngle number as 30 degrees, hydraulicDelay number as 0.2 seconds, offTrackingReduction number as 0.70 representing 70 percent reduction, isActive boolean as true. Implement method calculatePassiveOffTracking with parameters segmentLength and turnRadius: calculate using formula offTracking = segmentLength squared divided by (2 times turnRadius), return offTracking value. Implement method calculateActiveSteeringAngle with parameters upstreamAngle downstreamAngle turnRadius velocity: calculate passive off-tracking for this segment, multiply by (1 minus offTrackingReduction) to get compensated off-tracking, calculate required steering angle to achieve compensated off-tracking, constrain to maxSteeringAngle, return steering angle. This reduces off-tracking by 70 percent for segments with steerable axles which is critical for 13-19 axle configurations.

PART 4: SWEPT PATH SIMULATION ENGINE
STEP 12 - CREATE TURN SIMULATION CORE: Create class SweptPathSimulator with method simulateTurn taking parameters vehicle object, turnData object, simulationSteps number defaulting to 200. Initialize empty envelopePositions array. Calculate angle increment as turnData.angle divided by simulationSteps. Initialize tractorPosition at turnData.entryPoint with angle 0. Loop from step 0 to simulationSteps: calculate current angle as step times angle increment, update tractor position: x += wheelbase times cos(angle), y += wheelbase times sin(angle), set tractor angle to current angle, propagate through segments: for each segment after tractor calculate articulation angle from previous segment using ArticulationJoint, if segment is SteerableDolly calculate active steering angle using calculateActiveSteeringAngle, update segment position using updatePosition method with upstream position and articulation angle, get corners of each segment especially cargo segment, calculate envelope by finding min/max x and y from all corners, store envelope in envelopePositions array with current angle and all corner positions. Return simulation result object with envelopePositions array, maxWidth number, maxWidthAngle number, vehicle configuration used.
STEP 13 - IMPLEMENT CARGO ENVELOPE TRACKING: Create function calculateCargoEnvelope with parameters cargoSegment and allSegments array. Get cargo position from cargoSegment.position. Get cargo dimensions: length cargoSegment.length, width cargoSegment.width, height cargoSegment.height if available. Calculate lateral offset for cargo exceeding trailer width: offset = (cargo.width minus trailer.width) divided by 2. Compute four corners in local cargo coordinates: frontLeft as (-length/2, -width/2), frontRight as (-length/2, width/2), rearLeft as (length/2, -width/2), rearRight as (length/2, width/2). Apply rotation transformation using cargo angle: for each corner: rotatedX = cornerX times cos(angle) minus cornerY times sin(angle), rotatedY = cornerX times sin(angle) plus cornerY times cos(angle). Translate to world coordinates by adding cargo position. Calculate distance from turn center for each corner. Find maximum distance identifying widest point. Determine critical corner as the one with maximum distance typically front outer corner. Return envelope object with cornerPositions array of objects with x y, maxWidth number, criticalCorner string identifier, maxWidthLocation object with x y coordinates.
STEP 14 - CREATE OFF-TRACKING CALCULATOR: Create function calculateTotalOffTracking with parameters vehicle and turnRadius. Initialize totalOffTracking as 0 and contributions array as empty. Loop through each segment in vehicle.segments: if segment is passive (not SteerableDolly with steeringActive true) calculate passive off-tracking: segmentOffTracking = segment.length squared divided by (2 times turnRadius), add to totalOffTracking, push to contributions array with segment name and contribution value. If segment is SteerableDolly with steeringActive true: calculate passive off-tracking, apply reduction factor from segment.offTrackingReduction, add reduced amount to totalOffTracking, push to contributions showing original and reduced values. Sort contributions by value descending to identify main contributors. Return object with totalOffTracking number, contributions array, largestContributor string, reductionFromSteerable number showing how much was saved by steerable axles. Display warning notification if totalOffTracking exceeds 4 meters indicating extremely tight turn or very long vehicle.
STEP 15 - IMPLEMENT COLLISION DETECTION: Create function checkCollisions with parameters envelopePositions array and roadWidthProfile from turn detector. Initialize collisions array as empty. Loop through each position in envelopePositions: get angle and corners from position, find corresponding road width at this angle by interpolating roadWidthProfile, calculate road centerline position, for each corner calculate distance from road centerline, check if absolute distance exceeds road width divided by 2, if exceeds: corner is outside road boundary, calculate intrusion distance as absolute distance minus available width, create collision object with angle, corner identifier, intrusion distance, position coordinates, add to collisions array. After loop find first collision as minimum angle with collision. Find worst collision as maximum intrusion distance. Return collision report object with hasCollision boolean as collisions.length greater than 0, firstCollisionAngle number or null, firstCollisionCorner string, allCollisions array, worstCollision object with angle intrusion corner position, collisionCount number.
STEP 16 - CREATE CLEARANCE MARGIN CALCULATOR: Create function calculateClearanceMargins with parameters envelopePositions array and roadWidthProfile array. Initialize margins array as empty. Loop through each position in envelopePositions: get corners from position, find road width at this position angle, for each corner calculate distance to road edge: leftEdge distance and rightEdge distance, find minimum clearance among all corners at this position, classify clearance: if less than 0 meters STATUS_COLLISION, if less than 0.2 meters STATUS_CRITICAL, if less than 0.5 meters STATUS_WARNING, if less than 1.0 meters STATUS_CAUTION, else STATUS_SAFE. Store margin object with angle, minimumClearance number, status string, criticalCorner identifier. After loop find global minimum clearance. Calculate average clearance. Find sections with concerning clearance status. Return margins report with minimumClearance number, minimumClearanceAngle number, averageClearance number, criticalSections array filtered for WARNING or worse, clearanceProfile array with all margins, overallStatus string based on worst status found.
