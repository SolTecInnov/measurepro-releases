PART A — ALIGNMENT + PROFILE LINKED VIEWER (NEW FEATURE)

GOAL
Add road alignments to the Road Profile Viewer and save both objects, linked together. When the user moves a marker on the map (alignment view) the profile cursor moves to the matching station; when the user drags a cursor on the profile, the map marker moves to the matching station on the alignment.

CONCEPT
Use a shared linear reference “stationing” s (meters) along an alignment polyline.
Alignment is the polyline; Profile is altitude vs s.
Project GNSS points to the alignment to compute s and lateral offset.

DATA MODELS (Firebase + local cache)
1) Alignment
- id, projectId, name
- polyline: [{lat, lon}, ...]
- cumDistM: number[] (precomputed cumulative distance per vertex)
- createdAt, updatedAt, createdBy
2) Profile
- id, projectId, name
- alignmentId (required for linked mode)
- samples: [{ s_m, lat, lon, time, altitude_raw_m, altitude_selected_m, altitude_corrected_m, ... }]
- metadata: altitudeStrategy, altitudeOffsetM, axisMapping, buildVersion, createdAt

Persist BOTH in Firebase (project scope). Cache locally (IndexedDB). Support offline viewing.

CORE GEOMETRY FUNCTIONS
Implement these utilities (tested):
- projectPointToPolyline(pointLatLon, polylineLatLon, cumDistM) ->
    { s_m, offset_m, closestLatLon, segIndex, t }
  where s_m is station along polyline, offset is signed lateral distance (right positive), closest point is snapped point.
- pointAtStation(polylineLatLon, cumDistM, s_m) ->
    { lat, lon, bearingDeg }
- computePolylineCumDist(polylineLatLon) -> cumDistM

UI: PROFILE VIEWER SPLIT VIEW
Add a split view mode in Road Profile Viewer:
Left: Map with alignment polyline + a draggable station marker constrained to the alignment (snap as user drags).
Right: Profile graph with a vertical cursor line at station s.

INTERACTIONS (bidirectional)
- Map marker drag:
  -> snap marker to alignment
  -> compute s_m from snapped point
  -> update profile cursor to s_m
  -> show readout: station (km), lateral offset (m), corrected altitude at nearest sample
- Profile cursor drag:
  -> compute s_m from cursor x
  -> find pointAtStation(s_m) on alignment
  -> move map marker to that point
- “Link cursor” toggle (default ON)

PROFILE CREATION OPTIONS
- “Attach to alignment” for existing profiles: compute s_m per sample by projecting each sample lat/lon to the alignment.
- “Create alignment from driven path” (optional but useful):
  - create polyline from GNSS path (simplify with Douglas-Peucker)
  - save as alignment, then link profile

SAVE / LOAD
- List alignments and profiles in project
- Ability to load an alignment + linked profiles
- Save both as a “Linked Set” (alignmentId stored in profile)
- Ensure deleting an alignment warns if profiles reference it.

EXPORTS
- Export alignment as GeoJSON
- Export profile as CSV
- Export linked set as ZIP:
  - alignment.geojson
  - profile.csv
  - metadata.json (including calibration metadata)
