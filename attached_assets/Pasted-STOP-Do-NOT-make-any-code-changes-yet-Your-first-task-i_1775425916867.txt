--- BACKGROUND: WHAT IS BROKEN AND WHY ---

The POI action system has multiple problems:

PROBLEM 1 — IMMEDIATE TRIGGER (root cause)
In POISelector.tsx and KeyboardShortcutHandler.tsx, selecting a POI type
immediately calls a function that creates a log entry. This is wrong. The
measurement pipeline in useMeasurementLogging.ts already handles all logging
correctly. POI type selection should ONLY set the active type in the store —
nothing more. No log entry should ever be created at selection time.

PROBLEM 2 — DUPLICATE COUNTER DETECTION PATHS
There appear to be TWO separate code paths that can log a POI in
counterDetection mode simultaneously:
  (a) processCounterDetection() — called by a 150ms setInterval timer
  (b) A logBufferedPOI() function — inside the handleMeasurementChange useEffect
      that also handles counterDetection mode
If both paths are active at the same time, they will produce duplicate POI
entries for the same physical object. One of these paths must be removed or
disabled.

PROBLEM 3 — TWO SOURCES OF TRUTH FOR ACTIVE POI TYPE
Some functions read selectedPOIType (from props).
Some functions call getSelectedType() (from the POI store directly).
If these two sources ever diverge, measurements get logged with the wrong POI
type. All functions must use the same single source.

PROBLEM 4 — WRONG NOTE FORMAT IN ALL LOGGING PATHS
Multiple logging functions produce incorrect note text:
  - Some include "Visual documentation" text that must be removed
  - Some omit GND REF entirely or make it conditional on > 0
  - GND REF must ALWAYS appear in height-clearance POI notes, even when 0.00m
  - Measurement-free POIs should have only the POI type label in their note

PROBLEM 5 — MISSING PREREQUISITE GUARDS ON UI-TRIGGERED ACTIONS
auto-capture-no-measurement, open-manual-modal, and voice-note actions in
POISelector and KeyboardShortcutHandler have no guards. They must check:
  Guard A: an active survey exists
  Guard B: a logging mode is selected (not "manual" / "none" / "off")
  Guard C: isLogging === true (for auto-capture-no-measurement only)
If any guard fails, show a descriptive toast and do nothing else.

PROBLEM 6 — "NONE" POI TYPE DEFAULT IS WRONG
In poiActions.ts, DEFAULT_POI_ACTIONS maps '' (empty string) to
'auto-capture-and-log'. It should map to 'select-only'.
