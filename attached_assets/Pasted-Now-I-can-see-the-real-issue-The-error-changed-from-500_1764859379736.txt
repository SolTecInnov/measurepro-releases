The issue is MeasurePRO's proxy is NOT forwarding the Authorization header to RoadScope. When MeasurePRO calls /api/roadscope/proxy/surveys, the proxy needs to pass the Authorization: Bearer mpro_... header to RoadScope, but it's not doing that.

For MeasurePRO team - here's the fix:

In the proxy code, make sure to forward the authorization header:

// In MeasurePRO's proxy route handler
app.all('/api/roadscope/proxy/*', async (req, res) => {
  const path = req.params[0];
  
  const response = await fetch(`https://roadscope.app/api/measurepro/${path}`, {
    method: req.method,
    headers: {
      'Authorization': req.headers.authorization,  // <-- THIS IS MISSING!
      'Content-Type': 'application/json'
    },
    body: req.method !== 'GET' ? JSON.stringify(req.body) : undefined
  });
  
  const data = await response.json();
  res.status(response.status).json(data);
});

The key line is: 'Authorization': req.headers.authorization

Without this, the proxy sends requests to RoadScope without any authentication, causing the 401 Unauthorized error.
