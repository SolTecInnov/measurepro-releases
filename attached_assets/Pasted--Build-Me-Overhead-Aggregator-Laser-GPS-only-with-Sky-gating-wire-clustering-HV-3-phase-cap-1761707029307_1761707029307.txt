🔧 Build Me: Overhead Aggregator (Laser+GPS only) with “Sky” gating, wire clustering, HV 3-phase cap, and City mode
0) Goals (do this, nothing else)

Consume synchronized GPS and LDS-30 distance samples and emit minimal, useful POIs:

Bridges/gantries/trees → one POI: the lowest clearance under the object.

Utility wires (dense) → one POI per 10 m corridor (lowest only).

High-voltage lines → up to 3 POIs (the 3 lowest conductors), never more.

Use LDS-30 “sky/no-return” as a hard signal to segment events (start/stop).

Provide speed-aware presets (e.g., Highway/Suburban/City) and a “City mode” switch to aggressively suppress wire spam on ultra-dense streets (hundreds of drops).

Expose a clean TypeScript API (no UI) that Replit can call from elsewhere. Storage/logging is out of scope here—just return POIs.

1) Inputs and cadence

Call feed(sample) at 10–20 Hz with the fused sample:

type Sample = {
  t: number;                   // monotonic ms
  lat: number; lon: number;    // degrees
  headingDeg: number;          // 0..360
  speedKph?: number;           // GPS speed
  rawDistM?: number | null;    // raw LDS value in meters (may be null or near-max)
  status?: {                   // optional parsed LDS status
    noTarget?: boolean; overRange?: boolean;
  };
};


Normalize laser to distM:

If status.noTarget || status.overRange || rawDistM == null || rawDistM >= maxRange * skyThresholdFrac, then distM = null (treat as SKY).

Else distM = rawDistM.

2) Outputs (POIs)

Return POIs only at commit time:

type POI = {
  kind: 'bridge'|'gantry'|'tree_canopy'|'wires_util'|'wires_hv'|'overhead';
  distance_m: number;         // min clearance for this item (meters, 2 decimals)
  lat: number; lon: number;   // WGS84
  s_from_start_m: number;     // meters traveled during the event until the min
  t: number;                  // monotonic ms at min
  source: 'laser';
  tags?: string[];            // e.g., ['city_mode'] if City mode was active
};

3) Event model (state machine)

IDLE → waiting for start.

TRACKING → under an overhead object/line: collect minima and distance-along track.

HOLD → end candidate; wait brief persistence before commit.

Start condition (strict):

We were in SKY for at least skyMinBeforeStartMs.

We now have valid measurements persisting for solidMinToStartMs.

Drop vs baseline Δd = (baseline - distM) ≥ dStart.

End condition:

Either we return to SKY, or Δd ≤ dEnd, then wait tEndMs (HOLD).

Also cut if sTraveled >= winEventMaxM (safety stop).

Commit:

If current tag is bridge|gantry|tree_canopy|overhead → one POI: global minimum.

If wires_util → split series into corridors of winCouloirM meters along the path and take one lowest per corridor.

If wires_hv → per corridor, cluster by height with gap ≥ minSepWireM and keep the 3 lowest clusters only.

Note on tags: Tags are optional labels set by the host app (e.g., via Stream Deck). Implement both:

setPendingTag(tag) to apply to the next event.

setCurrentTag(tag) to apply to the current (TRACKING/HOLD) event.
If no tag is set, default to overhead.

4) Baseline and “Sky-aware” logic

Maintain a baseline representing “open sky” reading:

Do not update baseline while distM == null (SKY)—keep last value.

While under object (non-SKY), update baseline with a very slow EMA to avoid chasing dips:

baseline = (1-α_obj)*baseline + α_obj*distM, with α_obj very small (e.g., 0.03).

Optionally allow a slightly faster EMA when recently back to sky (e.g., α_sky = 0.08) to re-lock quickly.

Compute drop vs baseline: Δd = baseline - distM (0 if distM null).

5) Distance-along (s) and projection formula

Track sTraveled by summing haversine increments between consecutive samples:

haversine_m = 2R * asin( sqrt( sin²((lat2-lat1)/2) + cos(lat1)*cos(lat2)*sin²((lon2-lon1)/2) ) )
R = 6 371 000 m; angles in radians


Store s_from_start_m at the min point.

6) Wire algorithms (formulas)

Minima detection (for wires)

While TRACKING, append a point whenever:

New global minimum (distM < dMin - dMinEpsilon)

Or “local dip” with amplitude ≥ minDropForLocal.

Each point stored as:

type MinimaPoint = { s:number; d:number; t:number; lat:number; lon:number };


Corridor grouping

Sort series by s.

Partition into buckets where (s - s_base) ≤ winCouloirM; when exceeded, start new corridor.

Inside each corridor, keep only the lowest dip (min d).
(For HV we still need height clustering, see below.)

Height clustering (HV only)

Sort all points in corridor by d ascending.

Greedy 1-D clustering: start cluster with the lowest d; add points within minSepWireM of the last in cluster; else start a new cluster.

Represent each cluster by its lowest d.

Keep up to 3 lowest clusters per corridor.

7) Speed-aware presets and City mode

Implement three presets and a City mode (toggle) that overrides wire behavior:

Presets (auto-select by speedKph each feed):

Highway (> 60 km/h): faster end, longer window

tEndMs -= 50, winEventMaxM += 10

Suburban (15–60 km/h): nominal values

LowSpeed (< 15 km/h): slightly easier start, longer hold

dStart -= 0.05, tEndMs += 100

City mode (explicit toggle, independent of speed):

Goal: log far fewer wires in ultra-dense streets.

Effects when cityMode=true and tag is wires_util:

Increase corridor width: winCouloirM_city = max(winCouloirM, 20)

Increase local-dip threshold: minDropForLocal_city = max(minDropForLocal, 0.20)

Hard rate limit per 100 m: e.g., maxUtilLogsPer100m = 1

Optional min separation by distance: if last util POI is within minPOIDistM_city (e.g., 40 m) → suppress.

For wires_hv in City mode: still allow up to 3, but drop corridors where the lowest d is above a “not-interesting” threshold (e.g., ignoreIfAboveM = 9.0), configurable.

All values configurable; defaults in section 8.

10) Rate limiting and City mode enforcement

Keep a sliding distance window of the last 100 m (sum of haversine steps). Track how many util POIs emitted in that window. If ≥ maxUtilLogsPer100m, suppress additional util POIs until old ones fall outside the 100 m window.

Also apply minPOIDistM_city: if the distance from last util POI (haversine(lat,lon,lastUtil.lat,lastUtil.lon)) < threshold, suppress.

11) Edge cases & guards

If GPS fix is poor, allow logging but include a warning flag in diagnostics (host app decides what to do).

If distM oscillates quickly around thresholds, the HOLD hysteresis prevents chatter.

If an event runs longer than winEventMaxM, force commit with whatever minima collected.

If wires_hv lowest d > hvIgnoreIfAboveM and cityMode=true, drop the corridor entirely (not interesting enough in dense city context).

12) Test harness (include a minimal one)

Provide a tiny in-memory replay tool that feeds prerecorded samples to verify:

HV with 3 conductors → ≤ 3 POIs.

Target 10–20 Hz; all operations O(1) or O(n) per event (n = points in current event only).

No external dependencies except zod (optional) for config validation.
