3. Technical SEO checklist (implement fixes)
For the main public pages (homepage, features, pricing, contact, docs/FAQ if present):
1.	Head tags
•	Ensure each page has a unique, descriptive `<title>` (~55–65 chars) that includes “MeasurePRO”, key OS/OW / LiDAR terms, and “measure-pro.app”.
•	Ensure each page has a strong `<meta name="description">` (~140–160 chars) focused on OS/OW road surveys, LiDAR, clearances, and who it’s for.
•	Include `<meta name="robots" content="index, follow">` on all indexable pages.
•	Ensure viewport tag, charset, and language attributes are correct (`lang="en"` or bilingual if the content is actually bilingual).
2.	Canonical and indexing
•	Add `<link rel="canonical">` on each page pointing to the preferred URL (https + non‑www, e.g., `https://measure-pro.app/...`).
•	Make sure there is no accidental `noindex`, `nofollow`, or disallow` on important pages.
•	Create or correct `robots.txt` to:
•	Allow crawling of all important content pages.
•	Block only genuine non-public or duplicate URLs if needed (e.g., admin, test pages).
3.	Sitemap
•	Create/update `sitemap.xml` that includes only canonical, public URLs.
•	Ensure it’s accessible at `/sitemap.xml` and linked from `robots.txt`.
•	Include: homepage, features/product pages, pricing, contact/sales, documentation/FAQ, and any key marketing pages.
4.	Performance & Core Web Vitals
•	Optimize for fast load, especially on mobile: compress/resize large images, defer non-critical JS, remove unused scripts/styles if possible.
•	Make sure the landing pages (especially homepage) render main content quickly and are not blocked by heavy scripts.
•	Use lazy loading for below‑the‑fold images and non-critical components.
5.	PWA / technical integrity
•	Ensure service worker and manifest remain valid and do not interfere with indexing (e.g., avoid making the app shell a blank JS‑rendered page without content).
•	Ensure server responses for key pages return HTTP 200, not 3xx/4xx/5xx except where intentional (e.g., redirects from HTTP → HTTPS).
