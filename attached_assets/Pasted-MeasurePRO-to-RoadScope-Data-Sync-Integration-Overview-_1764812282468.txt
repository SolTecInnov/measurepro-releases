MeasurePRO to RoadScope Data Sync Integration
Overview
Implement a data synchronization system in MeasurePRO PWA to transfer survey data (POIs, routes, traces, images, videos, timelapse frames) to RoadScope using the MeasurePRO REST API.

Base URL: https://roadscope.app/api/measurepro (or development URL)

Authentication
API Key Management
MeasurePRO users must obtain an API key from RoadScope:

User logs into RoadScope with the same email used in MeasurePRO
User navigates to Admin > MeasurePRO API Keys
User generates an API key with required scopes
User copies the key (shown only once) and enters it in MeasurePRO settings
API Key Format
Prefix: mpro_
Length: 64 characters total
Example: mpro_a1b2c3d4e5f6...
Authentication Header
All API requests must include:

Authorization: Bearer mpro_your_api_key_here
Content-Type: application/json

Required Scopes
For full sync functionality, the API key needs these scopes:

surveys:read - List existing surveys
surveys:write - Create/update surveys
pois:read - Read POI data
pois:write - Create/update POIs
routes:write - Create/update routes/traces
files:upload - Get signed URLs for file uploads
files:read - Read file metadata

User captures POIs, routes, photos in the field

Implementation Logic
// Store linking preference in MeasurePRO's local storage
interface SurveyLink {
  measureproSurveyId: string;     // MeasurePRO's local survey ID
  roadscopeSurveyId: string;      // RoadScope's Firebase survey ID
  linkedAt: string;               // When the link was created
  syncMode: 'create_new' | 'link_existing';
}
// On first sync, user chooses:
async function initializeSurveyLink(measureproSurvey) {
  // Fetch existing RoadScope surveys
  const response = await fetch('/api/measurepro/surveys', {
    headers: { 'Authorization': `Bearer ${apiKey}` }
  });
  const { data } = await response.json();
  
  // Show selection UI to user
  const userChoice = await showSurveySelectionModal({
    measureproSurvey,
    existingSurveys: data.surveys
  });
  
  if (userChoice.mode === 'create_new') {
    // Create new survey in RoadScope
    const createResponse = await fetch('/api/measurepro/surveys', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        externalId: measureproSurvey.id,  // Link via externalId
        name: measureproSurvey.name,
        description: measureproSurvey.description,
        client: measureproSurvey.client
      })
    });
    const { data: newSurvey } = await createResponse.json();
    
    // Store the link
    saveSurveyLink({
      measureproSurveyId: measureproSurvey.id,
      roadscopeSurveyId: newSurvey.id,
      linkedAt: new Date().toISOString(),
      syncMode: 'create_new'
    });
    
    return newSurvey.id;
    
  } else {
    // Link to existing survey
    const selectedSurveyId = userChoice.selectedSurveyId;
    
    // Store the link (no API call needed, just remember the ID)
    saveSurveyLink({
      measureproSurveyId: measureproSurvey.id,
      roadscopeSurveyId: selectedSurveyId,
      linkedAt: new Date().toISOString(),
      syncMode: 'link_existing'
    });
    
    return selectedSurveyId;
  }
}
// On subsequent syncs, use stored link
async function syncPOIs(measureproSurvey, pois) {
  let roadscopeSurveyId = getSurveyLink(measureproSurvey.id)?.roadscopeSurveyId;
  
  if (!roadscopeSurveyId) {
    // First sync - show selection UI
    roadscopeSurveyId = await initializeSurveyLink(measureproSurvey);
  }
  
  // Sync POIs to the linked survey
  await fetch(`/api/measurepro/surveys/${roadscopeSurveyId}/pois`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      mode: 'upsert',
      pois: pois
    })
  });
}

Sync Strategy Implementation
Recommended Sync Flow
1. VALIDATE API KEY
   POST /auth/validate
   ↓
   Store userId and verify scopes
2. SYNC SURVEY
   POST /surveys
   {externalId: measurepro_survey_id, ...}
   ↓
   Store returned RoadScope survey ID
3. SYNC POIs (in batches of 500)
   POST /surveys/:surveyId/pois
   {mode: "upsert", pois: [...]}
   ↓
   Build externalId → RoadScope ID mapping
4. SYNC ROUTES/TRACES (in batches of 100)
   POST /surveys/:surveyId/routes
   {mode: "upsert", routes: [...]}
5. UPLOAD FILES
   a. Request signed URLs:
      POST /surveys/:surveyId/upload-urls
   
   b. Upload to signed URLs:
      PUT {signedUrl} with file data
   
   c. Register files:
      POST /surveys/:surveyId/files
6. VERIFICATION
   GET /surveys
   Compare counts with local data

Incremental Sync
For ongoing sync during field work:

// Track sync state in IndexedDB
const syncState = {
  surveyId: "roadscope_id",
  lastSyncTime: "2024-01-15T10:00:00Z",
  pendingPOIs: [],
  pendingRoutes: [],
  pendingFiles: []
};
// On POI creation/update
function queuePOIForSync(poi) {
  syncState.pendingPOIs.push(poi);
  if (navigator.onLine) {
    attemptSync();
  }
}
// Sync when online
async function attemptSync() {
  if (syncState.pendingPOIs.length >= 10 || timeSinceLastSync > 60000) {
    await syncPOIs(syncState.pendingPOIs);
    syncState.pendingPOIs = [];
    syncState.lastSyncTime = new Date().toISOString();
  }
}

Offline Handling
// Queue operations when offline
window.addEventListener('online', async () => {
  await syncQueuedOperations();
});
// Retry with exponential backoff
async function syncWithRetry(operation, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await operation();
    } catch (error) {
      if (error.status === 429) { // Rate limited
        await delay(Math.pow(2, i) * 1000);
      } else if (error.status >= 500) { // Server error
        await delay(Math.pow(2, i) * 1000);
      } else {
        throw error; // Client error, don't retry
      }
    }
  }
  throw new Error('Max retries exceeded');
}

Error Handling
Error Response Format
{
  "success": false,
  "error": "Error message",
  "code": "ERROR_CODE",
  "details": {}
}

Common Error Codes
Status	Code	Description	Action
400	INVALID_REQUEST	Missing/invalid fields	Check request body
401	UNAUTHORIZED	Invalid/expired API key	Re-authenticate
403	FORBIDDEN	Insufficient scopes	Request new API key
403	NOT_OWNER	Survey belongs to different user	Verify survey ID
404	NOT_FOUND	Survey/POI not found	Check IDs
409	DUPLICATE	ExternalId exists (create mode)	Use upsert mode
429	RATE_LIMITED	Too many requests	Wait and retry
500	SERVER_ERROR	Internal error	Retry with backoff
Validation Errors
POI coordinate validation:

Latitude must be -90 to 90
Longitude must be -180 to 180
Both values must be valid numbers
{
  "success": false,
  "error": "Invalid coordinates (lat must be -90 to 90, lng must be -180 to 180)"
}

Rate Limits
API Key validation: 10 requests/minute
Survey operations: 30 requests/minute
POI batch: 10 requests/minute (500 POIs each = 5000 POIs/min)
Route batch: 10 requests/minute
File URL requests: 20 requests/minute
File registration: 20 requests/minute
Security Considerations
Store API key securely - Use secure storage (Keychain/Keystore), not localStorage
Validate API key on app launch - Detect expiration early
HTTPS only - All requests must use HTTPS
Don't log sensitive data - Never log API keys or user data
Handle token expiration - Prompt user to generate new key when expired
