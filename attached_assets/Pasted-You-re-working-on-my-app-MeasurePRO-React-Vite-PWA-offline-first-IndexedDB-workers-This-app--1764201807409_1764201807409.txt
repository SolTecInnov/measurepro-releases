2.2 Track pending writes + last successful write

In the main thread (where you instantiate workers / log measurements), implement a storage health tracker, e.g. src/lib/survey/storageHealth.ts:

export type StorageHealth = {
  pendingWrites: number;
  lastSuccessfulWriteAt: number | null;
};

const health: StorageHealth = {
  pendingWrites: 0,
  lastSuccessfulWriteAt: null,
};

export function incrementPending(count: number) {
  health.pendingWrites += count;
}

export function applyAck(idsCount: number, committedAt: number) {
  health.pendingWrites = Math.max(0, health.pendingWrites - idsCount);
  health.lastSuccessfulWriteAt = committedAt;
}

export function getStorageHealth(): StorageHealth {
  return { ...health };
}


Hook this into the worker messages:

When sending a SAVE_MEASUREMENTS batch from main → logger/storage worker, call incrementPending(batch.length).

When the main thread receives MEASUREMENTS_SAVED_ACK from the storage worker, call applyAck(ids.length, committedAt).

export async function createSurveyCheckpoint(surveyId: string): Promise<void> {
  const db = await openSharedSurveyDB();

  // 1. Fetch last checkpoint for this survey (if any)
  const lastCheckpoint = await getLastCheckpointForSurvey(db, surveyId);
  const fromTimestamp = lastCheckpoint?.summary.lastMeasurementAt ?? null;

  // 2. Use existing export logic, but you can optimize:

4.3 Schedule checkpoints every ~10 minutes

In the part of the app that manages an active survey (e.g. SurveySessionManager / useActiveSurvey), add:

const CHECKPOINT_INTERVAL_MS = 10 * 60 * 1000; // 10 minutes

let checkpointTimer: number | null = null;

export function startSurveyCheckpointTimer(surveyId: string) {
  stopSurveyCheckpointTimer();
  checkpointTimer = window.setInterval(() => {
    void createSurveyCheckpoint(surveyId).catch(err => {
      console.error('[CHECKPOINT] Failed to create checkpoint', err);
    });
  }, CHECKPOINT_INTERVAL_MS);
}

export function stopSurveyCheckpointTimer() {
  if (checkpointTimer !== null) {
    clearInterval(checkpointTimer);
    checkpointTimer = null;
  }
}


Call:

startSurveyCheckpointTimer(currentSurveyId) when a survey is started / resumed.

stopSurveyCheckpointTimer() when survey ends or app navigates away.

Optionally, show “Last auto-backup: X min ago” in the UI by reading the latest checkpoint’s createdAt.

5. Flush pending batches on tab close / backgrounding

To further reduce exposure, ensure any pending batches are flushed when the app is minimized or closed:

In main thread initialization:

function requestFlush() {
  loggerWorker.postMessage({ type: 'FLUSH' } as FlushMessage);
  storageWorker.postMessage({ type: 'FLUSH' } as FlushMessage);
}

window.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'hidden') {
    requestFlush();
  }
});

window.addEventListener('beforeunload', () => {
  requestFlush();
});


In logger/storage workers:

On receiving FLUSH, immediately write any queued measurements and then send back FLUSH_ACK.

This keeps the “uncommitted in RAM/queue” window small even if the user closes the app.
