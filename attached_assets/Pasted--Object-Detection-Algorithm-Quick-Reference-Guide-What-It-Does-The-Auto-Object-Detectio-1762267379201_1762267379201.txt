# Object Detection Algorithm - Quick Reference Guide

## 🎯 What It Does

The **Auto Object Detection** system automatically logs overhead obstructions (bridges, power lines, etc.) by:
1. Continuously measuring distance with a laser sensor
2. Tracking the **minimum clearance height** as you pass under objects
3. Detecting when the object has **completely passed**
4. Automatically logging the obstruction with GPS coordinates

---

## 🔑 Key Concept

The algorithm uses a **10-count debounce system**:

```
Counter = 0  →  Object is PRESENT (actively detecting)
Counter = 10 →  Object has PASSED (ready to log or idle)
```

---

## 📊 The Algorithm in 3 Steps

### Step 1: Detect Object Presence
```
While passing under object:
  ✓ Valid laser readings within detection range
  → Counter stays at 0
  → Continuously update minimum height
```

### Step 2: Detect Object Exit
```
After passing object:
  ✓ 10 consecutive "clear" readings
  → Counter increments from 0 to 10
  → Minimum height is preserved
```

### Step 3: Log the Obstruction
```
When counter reaches 10:
  ✓ Create log entry with:
    - Minimum height detected
    - GPS coordinates (from minimum point)
    - Timestamp
    - Intensity
    - Notes/obstruction type
  → Reset for next object
```

## ⚙️ Configuration Parameters

| Parameter | Purpose | Typical Value |
|-----------|---------|---------------|
| **Distance Threshold** | Ignore readings too close (ground/vehicle) | 0.5 m |
| **Intensity Threshold** | Minimum signal strength for valid reading | 50 |
| **Base Height** | Sensor height above ground | 4-5 m |
| **Max Height** | Maximum detection range | 30 m |
| **Counter Max** | Consecutive clear readings needed | 10 |

---

## 🔍 Detection Conditions

### Counter INCREMENTS (No Object) When:

**ANY of these conditions:**
- ✓ Laser reads beyond max height (> 30m typical)
- ✓ Distance too close (≤ distance threshold) 
- ✓ Intensity too weak (≤ intensity threshold)

**AND:**
- ✓ A valid minimum height was previously recorded

### Counter RESETS (Object Present) When:

- ✓ Laser reading is within valid detection range
- ✓ Distance: threshold < distance < max height
- ✓ Intensity: above threshold

## 🧮 The Core Logic (Simplified)

```python
for each_laser_reading:
    
    # Update minimum if this is lower
    if reading_is_valid and reading < minimum:
        minimum = reading
        save_gps_coordinates()
        save_timestamp()
    
    # Determine if object is present
    if reading_indicates_no_object:
        counter += 1  # Count clear readings
    else:
        counter = 0   # Reset, object still there
    
    # Check if we should log
    if counter >= 10:
        log_entry(
            height = minimum,
            gps = saved_coordinates,
            timestamp = saved_timestamp
        )
        reset_minimum()
```

---

## 🚨 Why 10 Consecutive Readings?

The 10-count debounce ensures:

1. **Confirmation**: Object has truly passed (not just a gap)
2. **Stability**: System is in stable "no object" state
3. **Noise filtering**: Brief signal loss doesn't trigger false logs
4. **Time buffer**: ~0.1-0.5 seconds at typical speeds ensures clearance

## 🔧 How to Adjust Detection

### Make Detection More Sensitive:
- ⬇️ Decrease intensity threshold (accept weaker signals)
- ⬇️ Decrease distance threshold (detect closer objects)
- ⬇️ Lower counter max (log after fewer clear readings)

### Make Detection More Conservative:
- ⬆️ Increase intensity threshold (require stronger signals)
- ⬆️ Increase distance threshold (ignore closer readings)
- ⬆️ Raise counter max (require more clear readings)

---

## ⚠️ Important Notes

1. **Minimum is King**: The logged height is always the LOWEST point detected during the entire pass
2. **GPS Sync**: GPS coordinates are captured at the moment of minimum height (not at log time)
3. **Edge-Triggered**: Logs on the transition from "object present" to "object absent"
4. **One Log Per Object**: Each complete pass generates exactly one log entry
5. **Requires Motion**: System assumes you're moving - won't work if stationary

## 🐛 Common Issues & Solutions

### Problem: Not logging obstructions
**Check:**
- ✓ Mode set to "Auto Object Detection"
- ✓ Logging is started (not stopped)
- ✓ Intensity threshold not too high
- ✓ Object height within Base to Max range

### Problem: Logging too many entries
**Check:**
- ✓ Distance threshold not too low
- ✓ Laser not bouncing off vehicle/ground
- ✓ Counter max might need increase

### Problem: Missing low obstructions
**Check:**
- ✓ Base height set correctly
- ✓ Max height high enough
- ✓ Distance threshold not filtering valid readings

## 📊 Quick Math

```
Total Height = Laser Distance + Base Height

Example:
  Laser reads: 5.2 m
  Base height: 4.0 m
  ─────────────────
  Total height: 9.2 m ← This is the clearance above ground
```
